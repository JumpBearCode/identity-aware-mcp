# Multi-Client Implementation: Enabling Entra-Protected MCP to Be Accessed by Various Agent Clients (Document Index)

This folder contains the complete **exploration and implementation trajectory** of **"Multi-Client Access" for Identity-Aware MCP** — from "explaining OAuth principles for custom clients" to "using `/mcpproxy` to strip the `resource` parameter and connect non-VS Code clients such as Claude Code / opencode to Entra" (corresponding to the `client-explore` branch, merged into `main` via **PR #2** (`b666e87`)). The proxy implementation resides in `src/mcp-server/mcpproxy.py` + `main.py` (feature commit `1506ad8`, fine-tuning `21d4411`).

**Read in the order of the table below** — this follows both the chronological and logical sequence: understand principles and each client (#1) → write the first access plan (#2) → hit real-world issues (#3) → propose proxy solution and evaluate options (#4) → implement solution A (#5) → explain the integration principles (#6) → build a visual teaching app (#7). If you only want to see the **final working form**, jump directly to **#5**; if you want to **learn while watching animations**, start with **#7**.
