# action_bash Safety Guardrail: Output Redaction & Human Approval (Document Index)

This folder contains the complete **design and implementation trajectory** of **Identity-Aware MCP's "Action Gate Guardrail"** — from "designing a policy gate + human approval for `action_bash`" to "converging the current iteration scope and implementing output redaction with multi-client mandatory approval" (corresponding to the `policy-gate` branch and **PR #4**). Core implementation resides in `src/mcp-server/redact.py` + `main.py`, with client approval configuration in `.claude/settings.json`, `.vscode/settings.json`, and `opencode.json` (commit `568d4de`).

> **⚠️ Latest Conclusion (2026-07-18, per #5 final draft):**
> Empirical testing revealed that the post-exec redaction JSON field method can be trivially bypassed by **non-pure-JSON output** (`echo` headers, multi-command concatenation, tsv) as well as **scope evasion / jq field renaming** → **Post-exec redaction is hygiene, not a security boundary.**
> The final architecture converges to: **Boundary = identity least privilege** (`diagnose` = Reader, **zero data-plane**, cannot read secrets; privileged reads go through `action` = privilege escalation + human approval); **post-exec retains only Layer2 format regex, and only applies to `action_bash`**, used to mask account-level credentials such as storage keys / SAS / connection strings / JWT / PEM that **should not be echoed even when elevated**.
> Therefore, the **"three-layer redaction + all-tool activation" described in #3 has been superseded by #4 / #5**; #1–#4 are retained as **derivation process**.

**Read in table order** for the chronological and logical sequence: first establish a complete defense-in-depth blueprint (#1) → converge this iteration based on existing RBAC / oid-log capabilities and implement the initial redaction (#2, #3) → empirically disprove structural/entropy methods and polish the scrubber to its most stable v2 form (#4) → **cognitive inflection point: redaction is not a boundary, the boundary shifts to identity least privilege, post-exec reverts to Layer2-only (#5)**. **To see only the current finalized conclusion, jump directly to #5**; to see the complete derivation of why "field method / scope method / entropy method are all invalid," read #4.
