# ACA → Redis: Sandbox Execution Layer Design and Implementation (Document Index)

This folder contains the complete **design and implementation thread** of the **sandbox execution layer for Identity-Aware MCP** — from "moving the sandbox to ACA" to "using Redis for horizontal scaling coordination (distributed lock + Reaper leader election)". The corresponding code is on the `fix-redis` branch (feature commit `737e437`; see **#7**'s commit matrix for which code corresponds to which commit, whether it's deployed, and whether it's enabled).

**Read in the order of the table below** — this follows both the chronological and logical sequence: starting point (why / how to migrate to ACA) → design principles → selection rationale → implementation plan → measurement calibration → summary overview. If you only want to see the **final form**, jump directly to **#7**.
